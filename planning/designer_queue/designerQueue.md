## How This File Is Used

```
/spec-sync          Scans product_targets.md + Notion + feature specs
                    -> Identifies gaps (missing specs, open questions, TBDs)
                    -> Writes questions here, assigned to the owning designer

/designer-quiz      Designer runs this command
                    -> Reads this file, filters to their questions
                    -> Asks questions one-by-one interactively
                    -> Saves answers to raw_input/[designer]_[timestamp].md

/queue-review       Tim runs this command
                    -> Validates raw_input/ answers
                    -> Moves validated answers to clean_input/
                    -> Applies to feature spec files
                    -> Logs applied changes to output/
                    -> Updates question status here
```
